Core Concepts

Knowledge Items

A Knowledge Item is the fundamental unit in Agentic Local Brain. Each item represents a piece of collected knowledge with: title, content (stored as Markdown), source type, tags, summary, metadata, and vector embeddings. Types: note, bookmark, webpage, paper, email, file.

Collectors

Collectors are responsible for ingesting knowledge from various sources. Each collector:

  • Extracts content from its specific source type
  • Converts content to Markdown with YAML front matter
  • Generates metadata (word count, source info, dates)
  • Triggers tag and summary extraction

Available collectors: FileCollector (PDF, Markdown, text), WebpageCollector (web articles via httpx + readability), BookmarkCollector (browser bookmarks - Chrome, Firefox, Edge), PaperCollector (arxiv papers), EmailCollector (.eml and .mbox), NoteCollector (quick text notes).

Processors

After collection, content goes through processing:

Text Chunking

  • Splits large documents into smaller chunks for embedding
  • Configurable chunk size (default: 1000 chars) and overlap (default: 100 chars)
  • Preserves semantic boundaries (paragraph and sentence level)

Embedding Generation

  • Converts text chunks to vector representations
  • Supports DashScope (text-embedding-v4) and OpenAI-compatible providers
  • Used for semantic similarity search

Tag & Summary Extraction (Smart 3-Tier)

Tier 1: User-Provided → used directly
Tier 2: LLM Extraction → AI generates 3-5 tags + summary  
Tier 3: Built-in Algorithms → TF-IDF keywords + extractive summary

Storage

SQLite (Metadata)

Stores: item records, tags, tag-item associations, full-text search index (FTS5)

ChromaDB (Vectors)

Stores: vector embeddings for semantic search, document chunks with metadata

Query Modes

Keyword Search

  • Uses SQLite FTS5 full-text search
  • Fast, exact matching
  • Works without AI services

Semantic Search

  • Uses vector similarity (cosine distance)
  • Finds conceptually related content
  • Requires embedding service

RAG (Retrieval-Augmented Generation)

  • Combines semantic search with LLM
  • Retrieves relevant context, then generates answer
  • Requires both embedding and LLM services

Graceful Degradation

The system works at three levels:

  1. Full AI: All features (semantic search, RAG, LLM tagging)
  2. Embedding only: Semantic search + built-in tagging
  3. No AI: Keyword search + built-in tagging + extractive summary

Knowledge Mining (v0.6)

v0.6 introduces advanced knowledge mining capabilities that automatically discover patterns and relationships across your collected knowledge.

Knowledge Graph

  • Automatically extracts entities (people, technologies, concepts) from documents
  • Builds entity-relationship graphs with mention frequency tracking
  • Interactive graph visualization in the web interface

Cross-Document Relations

  • Discovers connections between documents via embedding similarity
  • Identifies shared entities across documents
  • Provides "related documents" for each knowledge item

Topic Clustering

  • Groups documents into topics using HDBSCAN clustering algorithm
  • LLM-generated topic labels for human-readable cluster names
  • Topic trend analysis with time-based visualization

Smart Recommendations

  • Personalized recommendations based on reading patterns
  • Time-decay weighted scoring for recency awareness
  • AI-generated recommendation reasons

核心概念

知识条目

知识条目是 Agentic Local Brain 的基本单元。每个条目代表一段收集的知识，包含：标题、内容（以 Markdown 存储）、来源类型、标签、摘要、元数据和向量嵌入。类型包括：笔记、书签、网页、论文、邮件、文件。

收集器

收集器负责从各种来源摄取知识。每个收集器：

  • 从其特定来源类型提取内容
  • 将内容转换为带 YAML 前置元数据的 Markdown
  • 生成元数据（字数、来源信息、日期）
  • 触发标签和摘要提取

可用收集器：FileCollector（PDF、Markdown、文本）、WebpageCollector（通过 httpx + readability 获取网页文章）、BookmarkCollector（浏览器书签 - Chrome、Firefox、Edge）、PaperCollector（arxiv 论文）、EmailCollector（.eml 和 .mbox）、NoteCollector（快速文本笔记）。

处理器

收集完成后，内容进入处理流程：

文本分块

  • 将大文档分割成较小的块以便嵌入
  • 可配置的块大小（默认：1000 字符）和重叠（默认：100 字符）
  • 保留语义边界（段落和句子级别）

嵌入生成

  • 将文本块转换为向量表示
  • 支持 DashScope（text-embedding-v4）和 OpenAI 兼容的提供商
  • 用于语义相似性搜索

标签与摘要提取（智能三层）

第一层：用户提供的 → 直接使用
第二层：LLM 提取 → AI 生成 3-5 个标签 + 摘要
第三层：内置算法 → TF-IDF 关键词 + 抽取式摘要

存储

SQLite（元数据）

存储：条目记录、标签、标签-条目关联、全文搜索索引（FTS5）

ChromaDB（向量）

存储：用于语义搜索的向量嵌入、带元数据的文档块

查询模式

关键词搜索

  • 使用 SQLite FTS5 全文搜索
  • 快速、精确匹配
  • 无需 AI 服务即可工作

语义搜索

  • 使用向量相似性（余弦距离）
  • 查找概念相关的内容
  • 需要嵌入服务

RAG（检索增强生成）

  • 结合语义搜索与 LLM
  • 检索相关上下文，然后生成答案
  • 需要嵌入服务和 LLM 服务

优雅降级

系统可在三个级别下工作：

  1. 完整 AI：所有功能（语义搜索、RAG、LLM 标签）
  2. 仅嵌入：语义搜索 + 内置标签
  3. 无 AI：关键词搜索 + 内置标签 + 抽取式摘要

知识挖掘 (v0.6)

v0.6 引入高级知识挖掘能力，自动发现已收集知识中的模式和关联关系。

知识图谱

  • 自动从文档中抽取实体（人物、技术、概念）
  • 构建实体关系图谱，跟踪提及频次
  • Web 界面中提供交互式图谱可视化

跨文档关系

  • 通过嵌入向量相似度发现文档之间的关联
  • 识别文档之间的共享实体
  • 为每个知识条目提供"相关文档"推荐

主题聚类

  • 使用 HDBSCAN 聚类算法将文档分组为主题
  • LLM 生成的主题标签，提供可读的聚类名称
  • 主题趋势分析，支持基于时间的可视化

智能推荐

  • 基于阅读模式的个性化推荐
  • 时间衰减加权评分，关注近期内容
  • AI 生成推荐理由
