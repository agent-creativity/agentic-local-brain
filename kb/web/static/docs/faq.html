FAQ & Troubleshooting

Frequently Asked Questions

What data does Agentic Local Brain collect?

Only the content you explicitly choose to collect. All data is stored locally on your machine. No data is sent to external services except for AI processing (embedding/LLM) when configured.

Does it work without an internet connection?

Yes, in degraded mode. Keyword search, built-in tag extraction, and file management all work offline. Semantic search and RAG require AI services.

Which AI providers are supported?

DashScope (Alibaba Cloud) and any OpenAI-compatible API (including Ollama for fully local AI).

How much disk space does it use?

Depends on your knowledge base size. Typical: 50-200MB for metadata + vectors for 1000 items. Original files are stored as Markdown.

Can I export my data?

Yes: localbrain export exports in Markdown or JSON format.

Is my data private?

All data stays on your local machine. AI API calls send only text content for processing. Use Ollama for fully local AI processing.

Troubleshooting

"Command not found: localbrain"

  • Ensure ~/.localbrain/bin is in your PATH
  • Run: export PATH="$HOME/.localbrain/bin:$PATH"
  • Add to ~/.zshrc or ~/.bashrc for persistence

Embedding/LLM service errors

  • Check API key: localbrain config show
  • Test connectivity: localbrain test embedding / localbrain test llm
  • Verify network access to the API endpoint

Web server won't start

  • Check port availability: lsof -i :8080
  • Try different port: localbrain web --port 9090
  • Check logs: ~/.localbrain/logs/

Slow search performance

  • Ensure ChromaDB is properly initialized
  • Run localbrain doctor for diagnostics
  • Consider adjusting chunk size for large documents

macOS Gatekeeper blocking binary

  • Run: xattr -cr ~/.localbrain/bin/localbrain

Performance Tips

  • Use background web server mode for persistent access
  • Configure appropriate chunk size for your content types
  • Regularly run localbrain doctor to check system health

常见问题与故障排除

常见问题

Agentic Local Brain 收集哪些数据？

仅收集您明确选择采集的内容。所有数据都存储在您的本地机器上。除非配置了 AI 处理（嵌入/LLM），否则不会将数据发送到外部服务。

没有互联网连接时能否使用？

可以，在降级模式下工作。关键词搜索、内置标签提取和文件管理都可以离线使用。语义搜索和 RAG 需要 AI 服务。

支持哪些 AI 提供商？

DashScope（阿里云）和任何兼容 OpenAI 的 API（包括用于完全本地 AI 的 Ollama）。

占用多少磁盘空间？

取决于知识库大小。典型情况：1000 个项目需要 50-200MB 用于元数据 + 向量。原始文件以 Markdown 格式存储。

我可以导出数据吗？

可以：localbrain export 以 Markdown 或 JSON 格式导出。

我的数据是否私密？

所有数据都保留在您的本地机器上。AI API 调用仅发送文本内容进行处理。使用 Ollama 可实现完全本地的 AI 处理。

故障排除

"Command not found: localbrain"

  • 确保 ~/.localbrain/bin 在您的 PATH 中
  • 运行：export PATH="$HOME/.localbrain/bin:$PATH"
  • 添加到 ~/.zshrc 或 ~/.bashrc 以持久化

嵌入/LLM 服务错误

  • 检查 API 密钥：localbrain config show
  • 测试连接：localbrain test embedding / localbrain test llm
  • 验证对 API 端点的网络访问

Web 服务器无法启动

  • 检查端口可用性：lsof -i :8080
  • 尝试不同端口：localbrain web --port 9090
  • 检查日志：~/.localbrain/logs/

搜索性能缓慢

  • 确保 ChromaDB 已正确初始化
  • 运行 localbrain doctor 进行诊断
  • 考虑为大型文档调整分块大小

macOS Gatekeeper 阻止二进制文件

  • 运行：xattr -cr ~/.localbrain/bin/localbrain

性能提示

  • 使用后台 Web 服务器模式以持久访问
  • 为您的内容类型配置适当的分块大小
  • 定期运行 localbrain doctor 检查系统健康状况
