User Question → Semantic Search → Context → LLM → Answer

Enhanced RAG Pipeline (v0.7)

The Enhanced RAG system implements a multi-stage retrieval pipeline for improved answer quality:

┌─────────────────────────────────────────────────────────────────────────┐
│                    Enhanced RAG Pipeline (v0.7)                          │
├─────────────────────────────────────────────────────────────────────────┤
│  1. Query Expansion                                                      │
│     └── Rewrite and expand queries for better recall                     │
│                                                                          │
│  2. Hybrid Retrieval (Keyword + Semantic with RRF)                       │
│     ├── Semantic Search → Top-K results                                  │
│     ├── Keyword Search → Top-K results                                   │
│     └── Reciprocal Rank Fusion → Combined ranking                        │
│                                                                          │
│  3. LLM Reranking                                                        │
│     └── Relevance scoring and result refinement                          │
│                                                                          │
│  4. Entity/Topic Enrichment                                              │
│     ├── Graph Query → Related entities                                   │
│     └── Topic Query → Related topics                                     │
│                                                                          │
│  5. Token-aware Context Assembly                                         │
│     └── Hierarchical context building with budget management             │
│                                                                          │
│  6. Answer Generation                                                    │
│     └── LLM synthesis with configurable prompt templates                 │
└─────────────────────────────────────────────────────────────────────────┘

Key components:

  • Query Expander: Rewrites queries to improve retrieval recall
  • Hybrid Retriever: Combines semantic and keyword search using RRF
  • Reranker: LLM-based relevance scoring for result refinement
  • Context Builder: Token-aware assembly with hierarchical formatting
  • Conversation Manager: Multi-turn context preservation
  • Prompt Templates: Configurable templates for different domains

Extension Points

Adding a New Collector

  1. Create new file in kb/collectors/
  2. Inherit from BaseCollector
  3. Implement collect(), _extract_content(), _generate_metadata()
  4. Export in kb/collectors/__init__.py
  5. Add CLI command in kb/cli.py

Adding a New Provider

The Embedder and TagExtractor support a provider pattern:

  • DashScope (default)
  • OpenAI-compatible (Ollama, vLLM, etc.)
  • Custom providers by extending the base classes

增强 RAG 流水线 (v0.7)

增强 RAG 系统实现了多阶段检索流水线，以提升回答质量：

┌─────────────────────────────────────────────────────────────────────────┐
│                    增强 RAG 流水线 (v0.7)                                 │
├─────────────────────────────────────────────────────────────────────────┤
│  1. 查询扩展                                                             │
│     └── 重写和扩展查询以提高召回率                                         │
│                                                                          │
│  2. 混合检索（关键词 + 语义，使用 RRF）                                    │
│     ├── 语义搜索 → Top-K 结果                                             │
│     ├── 关键词搜索 → Top-K 结果                                           │
│     └── 倒数排名融合 → 合并排序                                           │
│                                                                          │
│  3. LLM 重排序                                                           │
│     └── 相关性评分和结果优化                                               │
│                                                                          │
│  4. 实体/主题增强                                                         │
│     ├── 图谱查询 → 相关实体                                               │
│     └── 主题查询 → 相关主题                                               │
│                                                                          │
│  5. Token 感知上下文组装                                                  │
│     └── 分层上下文构建与预算管理                                           │
│                                                                          │
│  6. 答案生成                                                             │
│     └── 使用可配置提示词模板的 LLM 合成                                     │
└─────────────────────────────────────────────────────────────────────────┘

关键组件：

  • 查询扩展器：重写查询以提高检索召回率
  • 混合检索器：使用 RRF 结合语义和关键词搜索
  • 重排序器：基于 LLM 的相关性评分进行结果优化
  • 上下文构建器：Token 感知组装，支持分层格式
  • 会话管理器：多轮上下文保持
  • 提示词模板：支持不同领域的可配置模板

扩展点

添加新的收集器

  1. 在 kb/collectors/ 中创建新文件
  2. 继承 BaseCollector
  3. 实现 collect()、_extract_content()、_generate_metadata()
  4. 在 kb/collectors/__init__.py 中导出
  5. 在 kb/cli.py 中添加 CLI 命令

添加新的提供商

Embedder 和 TagExtractor 支持提供商模式：

  • DashScope（默认）
  • OpenAI 兼容（Ollama、vLLM 等）
  • 通过扩展基类创建自定义提供商
