Configuration Reference

Complete field-level reference for ~/.localbrain/config.yaml. Use the Test buttons on the Settings page to verify your LLM and Embedding configurations.

Full Example Configuration

# Data storage directory
data_dir: ~/.knowledge-base

# LLM configuration (for tagging, RAG, entity extraction, topic labels)
llm:
  provider: litellm            # litellm (recommended) | dashscope | openai_compatible
  model: dashscope/qwen-plus   # provider/model format for litellm
  api_key: ${DASHSCOPE_API_KEY} # env var reference or raw key
  base_url:                     # optional, for custom endpoints

# Embedding model (for semantic search, document similarity)
embedding:
  provider: dashscope           # dashscope | litellm | openai
  dashscope:
    model: text-embedding-v4
    api_key: ${DASHSCOPE_API_KEY}

# Knowledge Mining
knowledge_mining:
  entity_extraction:
    enabled: true
  topic_clustering:
    min_cluster_size: 5         # minimum docs to form a topic

# Text chunking
chunking:
  max_chunk_size: 1000          # max chars per chunk
  chunk_overlap: 100            # overlap between chunks

# Vector storage
storage:
  type: chroma
  persist_directory: ~/.knowledge-base/db/chroma

# Logging
logging:
  level: INFO                   # DEBUG | INFO | WARNING | ERROR
  log_dir: ""                   # empty = ~/.localbrain/logs/
  max_bytes: 10485760           # 10MB
  backup_count: 5

LLM Configuration

The LLM is used for: auto-tagging, RAG question answering, entity extraction (Knowledge Mining), and topic label generation.

FieldTypeDefaultDescription
llm.providerstringdashscopeProvider type. litellm (recommended), dashscope, or openai_compatible
llm.modelstringqwen-plusModel name. For litellm use provider/model format, e.g. dashscope/qwen-plus, openai/gpt-4o, anthropic/claude-3-5-sonnet-20241022
llm.api_keystring-API key. Supports ${ENV_VAR} format or raw key value
llm.base_urlstring-Custom API endpoint URL. Required for openai_compatible, optional for others. Use for Ollama (http://localhost:11434), vLLM, etc.

Knowledge Mining Configuration

Knowledge Mining automatically extracts entities, discovers document relationships, and clusters topics.

FieldTypeDefaultDescription
knowledge_mining.entity_extraction.enabledbooltrueEnable LLM-based entity extraction on new documents
knowledge_mining.topic_clustering.min_cluster_sizeint5Minimum documents required to form a topic cluster (HDBSCAN parameter)

Behavior

  • Auto-trigger on new documents: When a document is added, entity extraction, embedding generation, and relation discovery run automatically in a background thread (~14s per document)
  • Topic clustering: Global operation, not triggered per-document. Use the "Rebuild Topics" button on the Topics page or localbrain topics rebuild
  • Full pipeline CLI: localbrain mine run (incremental) or localbrain mine run --full (rebuild all)
  • Concurrency: localbrain mine run --workers 5 for parallel entity extraction

Chunking Configuration

FieldTypeDefaultDescription
chunking.max_chunk_sizeint1000Maximum characters per text chunk
chunking.chunk_overlapint100Character overlap between adjacent chunks

Larger chunks provide more context but less precise search. Smaller chunks are more precise but may lose context.

Graceful Degradation

The system works progressively based on available services:

Services AvailableFeatures
NoneFile collection, keyword search, manual tagging
Embedding only+ Semantic search, document similarity
LLM only+ Auto-tagging, summarization, entity extraction
Both+ RAG Q&A, full Knowledge Mining, smart recommendations

Testing Your Configuration

Two ways to verify your model configuration:

  1. Web UI: Go to Settings page, click "Test LLM Connection" or "Test Embedding Connection" buttons
  2. CLI: 
    # Check overall mining stats
localbrain mine stats

# Run a full pipeline to verify everything works
localbrain mine run

知识挖掘配置

知识挖掘自动提取实体、发现文档关系、聚类主题。

字段类型默认值说明
knowledge_mining.entity_extraction.enabledbooltrue是否在新文档入库时自动进行实体抽取
knowledge_mining.topic_clustering.min_cluster_sizeint5形成主题聚类的最少文档数（HDBSCAN 参数）

运行机制

  • 新文档自动触发：文档入库后，实体抽取、Embedding 生成、关系发现会在后台线程自动执行（每篇约 14 秒）
  • 主题聚类：全局操作，不按单篇触发。在主题页面点击"重建主题聚类"按钮，或执行 localbrain topics rebuild
  • 全量管道 CLI：localbrain mine run（增量）或 localbrain mine run --full（全量重建）
  • 并发加速：localbrain mine run --workers 5 并行实体抽取
