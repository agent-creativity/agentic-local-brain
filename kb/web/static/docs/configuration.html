Configuration

Configuration File

Location: ~/.localbrain/config.yaml

Created automatically by localbrain init

# Data directory
data_dir: ~/.knowledge-base

# Update server URL
update_server_url: http://localbrain.oss-cn-shanghai.aliyuncs.com

# Embedding model
embedding:
  provider: dashscope
  dashscope:
    model: text-embedding-v4
    api_key: ${DASHSCOPE_API_KEY}

# LLM configuration
llm:
  provider: dashscope
  model: qwen3.5-plus
  api_key: ${DASHSCOPE_API_KEY}

# Text chunking
chunking:
  max_chunk_size: 1000
  chunk_overlap: 100

# Storage
storage:
  type: chroma
  persist_directory: ~/.knowledge-base/db/chroma

# Logging
logging:
  log_dir: ""
  level: INFO
  max_bytes: 10485760
  backup_count: 5

Embedding Configuration

DashScope (Default)

  • Model: text-embedding-v4
  • Requires: DASHSCOPE_API_KEY environment variable or config setting
  • Get API key: https://dashscope.console.aliyun.com/

OpenAI-Compatible

For Ollama, vLLM, or any OpenAI-compatible API:

embedding:
  provider: openai
  openai:
    model: text-embedding-3-small
    api_key: your-key
    base_url: http://localhost:11434/v1  # for Ollama

LLM Configuration

DashScope (Default)

Supported models: qwen3.5-plus, qwen-plus, qwen-max

OpenAI-Compatible

llm:
  provider: openai
  model: gpt-4
  api_key: your-key
  base_url: https://api.openai.com/v1

Chunking Parameters

  • max_chunk_size: Maximum characters per chunk (default: 1000)
  • chunk_overlap: Overlap between adjacent chunks (default: 100)
  • Larger chunks = more context but less precise search
  • Smaller chunks = more precise but may lose context

Environment Variables

Variable Description Required
DASHSCOPE_API_KEY DashScope API key For AI features
OPENAI_API_KEY OpenAI API key If using OpenAI provider
KB_CONFIG_PATH Custom config file path No

Graceful Degradation

The system works without AI services:

  • No embedding service → keyword search only, built-in tag extraction
  • No LLM service → built-in extractive summary, TF-IDF tags
  • Both available → full semantic search, RAG, LLM tag extraction

Logging Configuration

  • Default log directory: ~/.localbrain/logs/
  • Log rotation: 10MB max, 5 backup files
  • Levels: DEBUG, INFO, WARNING, ERROR, CRITICAL

配置

配置文件

位置：~/.localbrain/config.yaml

由 localbrain init 自动创建

# 数据目录
data_dir: ~/.knowledge-base

# 更新服务器地址
update_server_url: http://localbrain.oss-cn-shanghai.aliyuncs.com

# 嵌入模型
embedding:
  provider: dashscope
  dashscope:
    model: text-embedding-v4
    api_key: ${DASHSCOPE_API_KEY}

# 大语言模型配置
llm:
  provider: dashscope
  model: qwen3.5-plus
  api_key: ${DASHSCOPE_API_KEY}

# 文本分块
chunking:
  max_chunk_size: 1000
  chunk_overlap: 100

# 存储
storage:
  type: chroma
  persist_directory: ~/.knowledge-base/db/chroma

# 日志
logging:
  log_dir: ""
  level: INFO
  max_bytes: 10485760
  backup_count: 5

嵌入模型配置

DashScope（默认）

  • 模型：text-embedding-v4
  • 需要：DASHSCOPE_API_KEY 环境变量或配置设置
  • 获取 API 密钥：https://dashscope.console.aliyun.com/

OpenAI 兼容

适用于 Ollama、vLLM 或任何 OpenAI 兼容的 API：

embedding:
  provider: openai
  openai:
    model: text-embedding-3-small
    api_key: your-key
    base_url: http://localhost:11434/v1  # 用于 Ollama

大语言模型配置

DashScope（默认）

支持的模型：qwen3.5-plus、qwen-plus、qwen-max

OpenAI 兼容

llm:
  provider: openai
  model: gpt-4
  api_key: your-key
  base_url: https://api.openai.com/v1

分块参数

  • max_chunk_size：每个分块的最大字符数（默认：1000）
  • chunk_overlap：相邻分块之间的重叠字符数（默认：100）
  • 更大的分块 = 更多上下文但搜索精度较低
  • 更小的分块 = 更精确但可能丢失上下文

环境变量

变量 描述 是否必需
DASHSCOPE_API_KEY DashScope API 密钥 AI 功能需要
OPENAI_API_KEY OpenAI API 密钥 使用 OpenAI 提供者时需要
KB_CONFIG_PATH 自定义配置文件路径 否

优雅降级

系统在没有 AI 服务时仍可工作：

  • 无嵌入服务 → 仅支持关键词搜索，使用内置标签提取
  • 无大语言模型服务 → 使用内置抽取式摘要，TF-IDF 标签
  • 两者都可用 → 完整的语义搜索、RAG、大语言模型标签提取

日志配置

  • 默认日志目录：~/.localbrain/logs/
  • 日志轮转：最大 10MB，5 个备份文件
  • 级别：DEBUG、INFO、WARNING、ERROR、CRITICAL
